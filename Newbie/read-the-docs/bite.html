Working on Your Own

It is impossible to learn and retain everything that Python can do. Reading the documentation, asking good questions, and learning how to craft search engine prompts are crucial skills for programmers. Both help you understand how to use libraries and functions correctly, troubleshoot issues, and discover new features.

Reading the Docs

It is important to have good resources where you can look up information. Some of the best information on Python comes are the Python Docs from python.org. The only downside is that some of the documentation can be difficult to understand for people who are new to programming in general. Like everything else, the more you engage in deliberate practice, the better you will get a reading and understanding documentation.

Asking Good Questions

A great website for asking questions is StackOverflow. In case you are not familiar, Stack Overflow is a website were people ask questions related to programming, and the community answers questions. The answers are voted on by the community. Generally, good and correct answers get upvoted. Often times, people have already asked the same question you have. You can search through the site and utilize the discussions others have already had.

Even if you choose to not use Stack Overflow directly, it is well worth your time to read their article on How do I ask a good question? or the blog post Writing The Perfect Question by Jon Skeet.

Crafting a Good Search Engine Prompt

As you continue your Python journey, search engines like Google become invaluable. Learning how to craft a good search prompt makes it that much easier to find solutions for the problems you are trying to solve.

The first tip is to make sure you are including important words that will narrow down search results. For example, the prompt format strings might include formatting strings in other programming languages. The prompt format strings python would be a better choice for our purposes.

Second, the top search engines offer various operators that give more control over search results. The first one we will discuss is quotes. If your first attempt didn't provide exactly what you are looking for, you can put parts of your search prompt in quotes. The prompt python newbies challenges includes lots of results that include the word beginner. While the prompt python "newbies" challenges returns results that include the phrase newbies.

The second operator we will discuss is the site operator. If you want to search a given site, for example Stack Overflow, from your search engine you can try a prompt like site:stackoverflow.com format strings python. All of the results will be from stackoverflow.com. If you want to exclude a site, place a hypen (-) in front. So, the prompt -site:stackoverflow.com format strings python will return results from all sites except stackoverflow.com

Finally, when you are dealing with errors you have never seen before, it can be helpful copy and paste them into a search engine. But copying and pasting the entire error message is not so helpful because it will contain information the only applies to you. Consider this code we first saw back in Newbie 45.
