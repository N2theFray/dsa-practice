This can be a complex concept to understand if you've never coded before so we'll look at what a loop is first. Programmatically, a loop is a process that, once complete, goes back to the beginning and starts again. There are different kinds of loops in programming, one of which happens to be the while loop.

Think of a while loop almost as a "consistent" if statement. While an if statement checks the condition just once and moves on, when a while loop is initiated it essentially monitors the condition and will keep running until it changes. Take some traffic lights for example.

With an if statement, we'd have to keep running the if statement over and over again to check the colour of the traffic light. E.g.:

if traffic_light == "red":
    print('Stop')
...

The status of the traffic light is not consistently checked by our if statement. We'd have to manually run the check every second to keep printing out that "Stop" string to ensure cars stay stopped.

A while loop however, will allow us to keep printing indefinitely until the condition changes:

while traffic_light == "red":
    print('Stop')

The format of the while loop is similar to that of an if statement:

  • We have the call to while along with the expression (the condition we're checking) next to it.
  • We then have our code indented by 4 spaces to show that it belongs "within" the while loop.
  • Execution wise, the while loop checks to see that traffic_light is assigned the string red and if it does, it'll print out the string Stop.
  • It'll then go back to the start, check to see if red is still assigned to traffic_light and keep going from there.

In plain English this while loop is saying: "While our traffic light is red, display the word 'Stop'".

The catch is that this while loop will go on forever. We have nothing in this current snippet of code that will tell it to stop. The traffic_light object isn't being changed so we'll get the 'Stop' message printed to the screen for eternity or until the system crashes!

This brings us to the second part of a while loop. When working with a while loop you'll generally have something in your code to get it to stop. The general way to do this is to change the condition. In our case, we'd change traffic_light to be "green" and that would break us out of the while loop as traffic_light would no longer be "red".

while traffic_light == "red":
    print('Stop')
    traffic_light = "green"
  • The while loop starts while traffic_light is "red".
  • It then prints "Stop".
  • traffic_light is then changed to "green".
  • The while loop checks to see if traffic_light is "red", sees that it is not "red" and stops running.

This is a pretty lousy while loop though as it'll only run once but you get the idea.

We can also just add the word break and when the while loop gets to it, it'll terminate:

while traffic_light == "red":
    print('Stop')
    break

This has the same effect of terminating the loop but doesn't change the traffic_light object to "green".

Another nifty trick is to use while True: to create an infinite loop. The True indicates existence. The whole truthiness thing. We're essentially saying, while this process exists, run our code.

This may seem like insanity but sometimes we actually do want code to be constantly executed and not stop. It's common to start with a while True loop then throw in a few if statements inside it.

A meaningful example is to have a basic "Main Menu" system written in text:

while True:
    print("1. Enter Game")
    print("2. Settings")
    print("3. Exit")
    choice = input("Enter a number between 1 and 3: ")
    if choice == "1":
        play_game()
    elif choice == "2":
        edit_settings()
    elif choice == "3":
        exit()
    else:
        print("Invalid entry, try again").

Now this may seem complex but all we're doing here is printing a menu with 3 options to screen, allowing a user to type in a menu option and then based on their choice, they get sent off somewhere else via another function (play_game(), edit_settings() or exit()).

The reason we have a while loop here is to ensure that once the user is finished with the function they were sent off to, they're looped BACK to the print statements that make up the Main Menu. While they're actually in their selected function, the while loop is waiting for them to come back. E.g.: Once you've finished editing settings, the while loop goes back to the start and prints out the 3 print statements again.
